# MangaWorkflow AI Model Training Guide

## ⚠️ CURRENT STATUS: MOCK MODE ONLY

**Important:** This project currently operates in **MOCK MODE**. Real YOLO model training is **NOT YET COMPLETED**.

### What This Means:
- The AI API returns **simulated/mock detection results**, not real model predictions
- Training scripts provided here are **scaffolds/placeholders** for future implementation
- All detection results, confidence scores, and bounding boxes shown in the UI are **generated by mock logic**
- No actual Manga109 dataset training has been performed
- The `best.pt` model file (if present) is a placeholder or untrained base model

### Why Mock Mode?
Mock mode allows the MangaWorkflow application to demonstrate the **complete AI workflow** (detect → review → accept → suggest → approve → execute) without requiring:
- Time-intensive model training (can take hours/days on GPU)
- Manual Manga109 dataset acquisition and preparation
- CUDA-compatible GPU hardware

### When to Use Real Training:
Real model training should be pursued when:
1. You have obtained the Manga109 dataset (requires academic access request)
2. You have prepared the dataset using `prepare_manga109.py`
3. You have access to a CUDA-compatible GPU
4. You are ready to invest significant time in model training and evaluation

## Overview
This guide explains how to train and evaluate the AI Region Segmentation models (e.g., YOLOv8-seg, U-Net) for the MangaWorkflow project. **Note: These instructions are for future implementation. Current system uses mock mode.**

## Requirements
* Python 3.9+
* CUDA-compatible GPU (Highly Recommended for real training)
* Prepared dataset (See `AI_DATASET_PREPARATION_GUIDE.md`)
* **Manga109 dataset must be manually requested and downloaded** (see Dataset Preparation Guide)

## Installation
1. Navigate to the `MangaWorkflow.AI` directory.
2. Create and activate a virtual environment:
   ```bash
   python -m venv .venv
   .venv\Scripts\activate
   ```
3. Install dependencies:
   ```bash
   pip install -r requirements-training.txt
   ```

## Running the API (Current Implementation)
The API server runs in **mock mode by default**. No trained model is required.

To run the API:
```bash
cd MangaWorkflow.AI
uvicorn api.main:app --reload --port 8001
```

The API will return mock detection results for demonstration purposes.

### Switching to Real Model (Future):
Once you have a trained model:
1. Copy your `best.pt` model to `MangaWorkflow.AI/models/yolo_seg/best.pt`
2. Update the API configuration to use "yolo" mode instead of "mock" mode
3. Restart the API server
